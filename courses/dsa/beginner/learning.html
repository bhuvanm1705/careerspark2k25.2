What is DSA?

Module 1

Data Structures and Algorithms (DSA) is the foundation of efficient programming. It dictates how we store data and how we solve problems.

Data Structures: Ways to organize data (e.g., Arrays, Trees).
Algorithms: Steps to solve a problem (e.g., Sorting, Searching).

Why it matters 
// Inefficient (O(n))
for x in data:
    if x == target: return True

// Efficient (O(1) with Hash Map)
return data.has(target)

Arrays & Strings

Module 2

An Array is a collection of items stored at contiguous memory locations. It's the simplest data structure.

Accessing an element by index is fast (O(1)), but inserting/deleting elements can be slow (O(n)) because you have to shift everything.

Array Traversal 
int[] arr = {10, 20, 30, 40};

// Loop through array
for(int i = 0; i < arr.length; i++) {
    print(arr[i]);
}

Linked Lists

Module 3

Unlike arrays, Linked List elements are not stored in contiguous locations. Each element (Node) points to the next.

Dynamic Size: Can grow or shrink.
Efficient Insertions: Easy to add/remove if you have the pointer.

Node Structure 
class Node {
    int data;
    Node next;
    
    Node(int d) { data = d; next = null; }
}

Stacks & Queues

Module 4

Stack: LIFO (Last In First Out). Like a stack of plates. Push/Pop.
Queue: FIFO (First In First Out). Like a line at a store. Enqueue/Dequeue.

Time Complexity (Big O)

Module 5
